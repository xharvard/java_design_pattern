1.1 Oracle产品系列
	3个主要的产品组合：数据库服务器，应用服务器，企业管理器
	
	处理模型：客户端-服务器（通信协议：Oracle Net）
	客户端层：用户，用户进程
	服务器层：执行SQL的服务器进程，实例，数据库本身
	
1.2 Oracle概念
	Oracle Database Server实例和数据库组成。
	常用工具：OUI,DBCA,SQL*Plus,SQL Developer
	
1.3 单实例体系结构（分布式体系结构属于OCM范畴）：一个实例打开一个数据库
	实例由内存和进程组成。暂存于RAM和CPU中。当关闭实例时，实例将消失。
	数据库由磁盘上的物理文件组成，这些文件一直存在。
	构成实例的进程称为后台进程。
	在操作系统提供的共享内存段实现的内存结构称为系统全局区（system global area,SGA）。
	SGA在实例启动时分配，关闭时释放。
	与每个服务器进程相关的是一个非共享内存区（程序全局区：pragram global area,PGA），此区是会话专用。
	oracle数据库的物理结构：数据文件，重做日志文件和控制文件。
	数据字典包含用来描述整个数据库的元数据。数据库的物理结构与逻辑结构之间的关联在此中定义和维护。
	数据文件：实际的数据存储文件。
	重做日志文件：对应用于数据的所有变更向量（DML:insert,update,delete）的一系列记录。用来防止数据丢失和还原。
	控制文件：存储数据库物理结构的详细信息，并且是连接到逻辑结构的起点。

	分布式体系结构：实例和数据库的多种组合
		实时应用群集（Real Application Clusters,RAC）：多个实例打开一个数据库
		流：多个oracle服务器相互之间传播事务
		数据保护：主数据库更新备用数据库
		整合上述组合，可以实现100%正常运行时间和0%数据丢失的目标，获得无限扩展性和性能。
		
		
	怎么确定数据库是单实例还是分布式系统的一部分：
	1，用SYSTE用户连接到数据库
	2，select parallel from v$instance;如果是单实例数据库，将返回NO。
	3，确定数据库是否通过备用数据库的保护来防止数据丢失：
		select protection_level from v$database;如果未受保护，返回UNPROTECTED。
	4，确定是否配置了流：
		select * from dba_streams_administrator;如果未配置，将不会返回任何行。
		
1.4 实例内存结构
	SGA结构：必需的有 数据库缓冲区缓存，日志缓冲区，共享池
	                     可选的有 大池，Java池，流池

	数据库缓冲区缓存：Oracle用来执行SQL的工作区域。可以动态调整大小，也可以对其自动管理。
	日志缓冲区：小型的，用于短期存储将写入磁盘上的重做日志的变更向量的临时区域。一般采用默认大小即可。实例启动时大小被固定，无法对其自动管理。
	共享池：有许多子结构，子结构由Oracle服务器内部管理。共享池可以动态调整大小，也可以对其自动管理。
		主要的4个组件：库缓存，数据字典缓存，PL/SQL区，SQL查询和PL/SQL函数结果缓存
		库缓存：内存区域，按其已分析的格式存储最近执行的代码。极大的提高性能。
		数据字典缓存：存储最近使用的对象定义：表，索引，用户和其他元数据定义的描述。
		PL/SQL区：存储的对象是过程，函数，打包的过程，打包的函数，对象类型定义和触发器。
		SQL查询和PL/SQL函数结果缓存：将查询的结果存储在内存中，再下次查询是，可以从缓存检索结果，而不是运行该查询。
		
		大池：供共享的服务器进程使用。可以动态调整大小，也可以对其自动管理。
		Java池：需要在数据库中运行Java存储过程时才需要，用作实例化Java对象所需的堆空间。可以动态调整大小，也可以对其自动管理。
		流池：Oracle流使用。可以动态调整大小，也可以对其自动管理。
